# SPO 参数配置指南

## 参数分工

现在参数分为两类：

1. `config.json`
   - 放全局计算参数
   - 用户直接编辑这个文件即可
2. `input/<scene>/depth_opt.py`
   - 放当前具体光场场景的几何参数
   - 视差范围仍然从这里读取，不放到 `config.json`

当前 `config.json` 默认内容：

```json
{
  "scale": 1.0,
  "bins": 64,
  "nD": 64,
  "sigma": 0.3,
  "guided_filter_radius": 10,
  "guided_filter_eps": 0.0001,
  "use_gpu": true
}
```

## 需要在 `depth_opt.py` 中配置的参数

这些参数与具体数据集绑定：

- `disp_min`: 场景最小视差
- `disp_max`: 场景最大视差
- `NumView`: 每个维度的视角数

## `config.json` 参数说明

### 1. `scale`

作用：控制 SPO 权重窗口的宽度。

建议：

- 纹理丰富、细节多：`0.9 ~ 1.2`
- 一般场景：`1.2 ~ 1.5`
- 平滑区域多、噪声偏大：`1.5 ~ 2.0`

现象判断：

- 深度边缘发糊：减小 `scale`
- 深度噪点较多：增大 `scale`

### 2. `bins`

作用：控制颜色直方图的离散精度。

建议：

- 颜色和纹理变化丰富：`128 ~ 256`
- 一般场景：`64 ~ 128`
- 噪声较大或颜色单调：`16 ~ 64`

现象判断：

- 颜色信息没有充分利用：增大 `bins`
- 对噪声过于敏感：减小 `bins`

### 3. `nD`

作用：深度标签数量，直接影响深度分辨率和计算量。

建议：

- 快速测试：`64`
- 常规使用：`96 ~ 128`
- 更高精度：`160 ~ 256`

现象判断：

- 深度图有明显阶梯感：增大 `nD`
- 运行太慢或显存不足：减小 `nD`

这是最影响计算时间的全局参数之一。

### 4. `sigma`

作用：控制水平 EPI 和垂直 EPI 的可靠性融合强度。

建议：

- 水平或垂直方向纹理非常明显：`0.2 ~ 0.3`
- 两个方向纹理比较均衡：`0.4 ~ 0.7`
- 纹理方向性弱、希望更稳：`0.7 ~ 1.0`

现象判断：

- 某一方向纹理主导很强：保持较小 `sigma`
- 想让两个方向融合更平稳：适当增大 `sigma`

### 5. `guided_filter_radius`

作用：guided filter 的空间半径，影响后处理平滑范围。

建议：

- 保留细节和边缘：`5 ~ 8`
- 一般场景：`10 ~ 12`
- 更强平滑去噪：`15 ~ 20`

现象判断：

- 深度图边缘被抹平：减小 `guided_filter_radius`
- 深度图噪点或孔洞较多：增大 `guided_filter_radius`

### 6. `guided_filter_eps`

作用：guided filter 正则项，影响边缘保持与平滑强度的平衡。

建议：

- 更强边缘保持：`1e-5 ~ 5e-5`
- 一般场景：`1e-4 ~ 5e-4`
- 更强平滑：`1e-3 ~ 1e-2`

现象判断：

- 边界需要更锐利：减小 `guided_filter_eps`
- 边界毛刺或噪点较多：增大 `guided_filter_eps`

### 7. `use_gpu`

作用：是否优先启用 GPU。

说明：

- 设为 `true`：优先尝试 CuPy
- 设为 `false`：强制使用 CPU
- 即使设为 `true`，若环境中没有 CuPy，程序也会自动回退到 CPU

## 推荐起始配置

### 通用场景

```json
{
  "scale": 1.4,
  "bins": 128,
  "nD": 128,
  "sigma": 0.5,
  "guided_filter_radius": 10,
  "guided_filter_eps": 0.0001,
  "use_gpu": true
}
```

### 高纹理场景

```json
{
  "scale": 1.0,
  "bins": 256,
  "nD": 192,
  "sigma": 0.3,
  "guided_filter_radius": 7,
  "guided_filter_eps": 0.00005,
  "use_gpu": true
}
```

### 低纹理或高噪声场景

```json
{
  "scale": 1.8,
  "bins": 64,
  "nD": 128,
  "sigma": 0.7,
  "guided_filter_radius": 16,
  "guided_filter_eps": 0.0005,
  "use_gpu": true
}
```

### 快速测试

```json
{
  "scale": 1.2,
  "bins": 64,
  "nD": 64,
  "sigma": 0.3,
  "guided_filter_radius": 7,
  "guided_filter_eps": 0.001,
  "use_gpu": false
}
```

## 推荐调参顺序

1. 先在 `depth_opt.py` 里把 `disp_min` / `disp_max` / `NumView` 设对
2. 再在 `config.json` 里调 `nD`
3. 然后按效果调 `scale` 和 `sigma`
4. 最后再调 `guided_filter_radius` 和 `guided_filter_eps`

这套顺序更稳，因为如果视差范围一开始就错了，后面再调全局参数也很难得到正常结果。
